(WIP)This feature is currently partially implemented and will eventually be part of the **Living Books** framework mod. For now, you can modify the existing book, but you can't add new ones yet.


--
This File is for guide only, do not modify
--

1_ Download or convert your book (https://cloudconvert.com/ Recomended)
2_ Convert to Json (ex: https://pdfjar.com/pdf-to-json)  (With Structured Format)
3_Import your Json File into "42\media\books" folder

--
4_ Copy or modify the Json to match the following format (usually only author is missing)

{
  "filename": "Book Title", -- Recomended --
  "author": "Book Author", -- Recomended --
  "pageCount": Numbers, -- Not mandatory --
  "pages": [
    {
      "pageNumber": 1,  -- Not mandatory --
      "headings": [
        {
          "text": "First Heading",      -- Big font, used for Chapters, Not mandatory --
          "level": 2
        }
      ],
      "paragraphs": [
        "Book Text"
      ]
   },  --> There must be no spaces, and there must always be a comma between paragraphs if you want to add/modify anything
    {
      "pageNumber": 2,
      "headings": [],
      "paragraphs": [         --> Next paragraph section with or without headings 
        "Text"
        ]
    },
}   --> End it this way.


EXTRA/TIPS:
The Enemy Lua and ZedScripts extensions for VS (recommended on the official wiki) will help you identify potential writing and syntax errors. 
If you want to insert a line break, use \n (e.g., World\nWord). 
However, try not to use it in the title or author fields, as it may cause visual bugs.
Don't worry about the book's length. Since it's just plain text, it's not very resource-intensive and can handle books of up to 2,000 pages without any noticeable performance impact.
If you want to see working examples right away, it's recommended to download/install one from the Workshop.
The fonts used are the game's default fonts. No custom fonts are used because making them compatible and readable by the game's engine requires a significant amount of work. It's not as simple as importing a `.ttf` file, at least not as far as I know.
