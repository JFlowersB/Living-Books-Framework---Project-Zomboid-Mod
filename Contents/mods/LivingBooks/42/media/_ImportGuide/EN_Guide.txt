Important:
- Everything inside the LivingBooksFramework "_ImportGuide/files" folder must be copied into your new AddonBookMod.
- Create your mod using the Build 42 folder structure (https://pzwiki.net/wiki/Mod_structure), or simply copy your ModTemplate into your Zomboid\Workshop folder.
- All files referenced in this guide are already included in the "_ImportGuide/files" folder. Just copy them and edit them as explained below.

Before you begin:
- Copy "LivingBooksItems.txt" into your AddonBookMod here:
  42/media/scripts/LivingBooksItems.txt
- Edit or duplicate the item blocks inside the file if you want to add more books. You only need to change the item name (leave the module and properties untouched). Watch out for syntax errors.
- Download or convert your book (https://cloudconvert.com/) [Free, no account required].
- Convert the file to JSON (for example: https://pdfjar.com/pdf-to-json) using the structured format. [Free, no account required].
- Place the generated JSON file into:
  42/media/books
- Create a file called "books.txt" inside the same folder:
  42/media/books/books.txt
- Add the full filename of your JSON book on a new line (example: mybook.json).
- Fill in the "itemFullType" field using the item name you created in "media/scripts/LivingBooksItems.txt" (Module.ItemName).
- Edit or fix the JSON so it matches the following format (add the author if it isn't already included).

WARNING:
- The comments (-- text --) are only there to guide you. Remove them once you're done.

{
  "filename": "Book Title",                        -- REQUIRED --
  "itemFullType": "Module.ItemName",               -- REQUIRED! Must match the item defined in "42/media/scripts/LivingBooksItems.txt" (this file must be included in your addon) --
  "author": "Book Author",                         -- Optional --
  "pageCount": Number,                             -- Optional --
  "pages": [
    {
      "pageNumber": 1,                             -- Optional --
      "headings": [
        {
          "text": "Chapter Title",                 -- Large font, intended for chapter headings. Optional --
          "level": 2
        }
      ],
      "paragraphs": [
        "Book text"
      ]
    },                                             --> Do not leave empty spaces, and always separate pages or paragraphs with commas.
    {
      "pageNumber": 2,
      "headings": [],
      "paragraphs": [                              --> Next page, with or without headings.
        "Book text"
      ]
    }
}

--> The JSON should end exactly like this.

Final tips:
- To insert a line break inside a paragraph, use \n (example: Word\nWord). However, avoid using it in the title or author fields, as it may cause visual issues.
- If you add more than one paragraph or page, always separate each element with a comma, and never leave empty entries.
- The Enemy Lua and ZedScript extensions for Visual Studio Code (recommended on the official wiki) can help you detect syntax and typing errors.
- Don't worry about book length. Since books only contain text, LivingBooks can handle books with over 2,000 pages with virtually no performance impact.

Remember:
- Each content pack contains exactly ONE book. If you want to add multiple books, create several small content-pack mods, or make a dedicated multi-book pack.
- "itemFullType" is the ONLY thing linking this JSON to an in-game item. LivingBooks scans every activated mod for media/books/book.json files when the game starts and matches them with the inventory item whose FullType is an exact match.
- If you'd like to see complete examples, download any of my compatible books from the Steam Workshop.

That's it!
Activate your AddonBookMod alongside LivingBooksFramework and enjoy a fully integrated reading experience!
